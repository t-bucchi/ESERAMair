# Backup to Flash

## 概要

ESERAMair の SRAM 全域 `1MB` を Pico の Flash 内に保存したり、Flash から SRAM へ復元することができます。

MSX実機のリセットスイッチで操作できるので、リセットスイッチ付きのMSXがあれば PC が無くても WiFi が無くてもSRAM状態の保存/復帰ができます。

似非RAMディスクの性質上、暴走したりするとSRAMのデータが壊れる可能性がありますが、Nextorドライバや初期ファイルがある状態をFlashに保存しておくことで、簡単にNextor環境を復元させることができます。

## RESET 操作による保存復帰

MSX の RESET スイッチ長押しで保存または復元を選択できます。

MSXの電源が入った状態で、RESET スイッチを押すと以下のような動作となります。

- RESET スイッチ押下から5秒未満 → 変化なし
- 5秒以上押す
  - LEDが低速点滅する
  - この状態でRESETを離す → FlashからRestore
- 10秒以上押す
  - LEDが高速点滅に変わる
  - この状態で離す → FlashへBackup

Backup/Restore 処理が開始されるとLEDが高速点滅します。
処理をしている間、MSXも動作が止まるはずです。
Backup/Restore が完了すると MSX が動作を開始し、
LEDも元のWiFiステータスの表示に変わります。

## コンソール操作

`backup` コマンドは通常コンソールから実行できます。

- `backup save`
  - SRAM 全域 `1MB` を Flash バックアップ領域へ保存します。
  - Flash は `4KB` セクタ単位で消去し、`256B` ページ単位で書き込みます。
- `backup restore`
  - Flash バックアップ領域を SRAM 全域へ復元します。
  - 未初期化のバックアップ領域でも確認なしに実行されます。
- `backup erase`
  - Flash バックアップ領域を全消去します。

## 注意事項

- バックアップ領域には整合性確認がないため、途中で電源断した場合の検出はできません。
- `backup restore` は保存済みデータの有無を判定しません。
- Flash 書き込み寿命があるため、高頻度の自動保存用途には向きません。
- REST API は追加されていません。操作はコンソールまたは RESET 長押しで行います。
